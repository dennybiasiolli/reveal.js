WIP Debugging Olympics

Funny Bugs and How We Fixed Them

Python (Django) & JavaScript (Vue.js)

Welcome everyone! Today, we'll take you through the “Debugging Olympics”—a fun, story-driven journey through real (and exaggerated) bugs at the backend/frontend interface.

Introduction

  • Why backend/frontend collaboration matters
  • What are the Debugging Olympics?
  • Get ready for fun, learning, and audience participation!
We'll show how teamwork, communication, and humor can turn debugging into a learning opportunity. Feel free to participate—there will be polls and a challenge round!

Setting the Stage

  • Each round = a real-world bug
  • Scripts, banter, and lessons
  • Audience: help us debug!
Each round is a real bug we've encountered. We'll act out the debugging process, share lessons, and invite you to join in.

Round 1: The Mysterious 500 Error

  • Frontend gets a 500 error
  • Backend logs are clean
  • What's going on?
A: “Hey, why is your frontend throwing a 500 error? My Django logs are clean as a whistle!” B: “Don't look at me! I'm just calling /api/dataz like you said.” A: “Wait… /api/dataz? It's /api/data! You added a ‘z'!” B: “Oops. So the real bug was… a typo?” A: “And the real lesson: Always double-check your endpoints—and maybe add some error handling!”

Round 2: The Disappearing Data

  • Data saved, but not showing up
  • Backend returns snake_case, frontend expects camelCase
B: “I just saved a new user, but nothing shows up in the list!” A: “The backend's returning the data. See? user_name and user_email.” B: “But my Vue app expects userName and userEmail! Snake_case vs. camelCase strikes again.” A: “Let's add a serializer to match the frontend's format.” B: “Lesson learned: Consistent data formats save headaches!”

Round 3: The Infinite Spinner

  • Loading spinner never stops
  • Backend response missing a field
B: “Why is my loading spinner spinning forever?” A: “Let's check the network tab… Oh, the backend forgot to include the status field in the response.” B: “My code waits for status before hiding the spinner!” A: “Fixing the response now. Spinner, be gone!” B: “Lesson: Always agree on API contracts!”

Round 4: The Time Traveler's Bug

  • Frontend displays wrong time
  • UTC vs. local time confusion
B: “Why does my app say the meeting is at 3am?” A: “Django stores times in UTC. Did you convert to local time in Vue?” B: “Nope. Let's use date-fns to fix that.” A: “Timezones: the real final boss of programming.”

Round 5: The Phantom CORS Error

  • Frontend can't fetch data
  • CORS misconfiguration
B: “I can’t fetch anything—CORS error!” A: “Let me check my Django CORS settings… Oops, forgot to add your frontend’s URL.” B: “Try now… It works! CORS: Can’t Operate, Requests Stopped.” A: “Lesson: Don’t forget your CORS headers!”

Round 6: The Vanishing Session

  • Users logged out unexpectedly
  • Session/cookie mismatch
B: “Why am I logged out every time I refresh?” A: “Session cookie settings—looks like the domain or secure flag is off.” B: “Let’s align our settings. Now it works!” A: “Authentication: fragile, but fixable.”

Round 7: The Language Barrier

  • Error message format mismatch
  • Frontend expects { error: 'message' }, backend sends { detail: 'message' }
B: “My error handler expects { error: 'message' }, but I get { detail: 'message' } from Django.” A: “Let’s standardize our error responses.” B: “Now my UI can actually show helpful messages!” A: “Lesson: Consistent error formats = happy users.”

Round 8: The Overzealous Linter

  • Frontend build fails due to linter
  • Backend-generated code doesn’t comply
B: “My build fails—Prettier hates your JSON output!” A: “Let's tweak the backend output or relax the linter rule.” B: “Now everyone's happy—including the linter.” A: “Lesson: Tooling should help, not hinder.”
